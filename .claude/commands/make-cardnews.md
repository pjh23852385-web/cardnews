# /make-cardnews

**주 채널**: 텔레그램 그룹방 (그룹 ID `-5041725581`)
**이 커맨드**: 텔레그램 쓰지 못하는 상황의 **백업 실행 모드**

---

## 사용법

```
/make-cardnews sources/2026-04/01. 카드뉴스 소스.md
```

인자: 소스 .md 파일의 상대/절대 경로

---

## 실행 흐름

3봇 팀이 순차 실행됨. 단, 발화는 콘솔 출력. 체크포인트는 콘솔에서 답변 받음.

1. **소스 읽기 + 파싱**
   - frontmatter `notion_url:` 추출 (있으면)
   - 본문 추출
   - `sources/YYYY-MM/<파일>.md` 위치 확인

2. **편집장** (`editor` 에이전트) — 오디언스 확인
   - 콘솔 출력: *"이 콘텐츠 누구한테 보여줄 거에요?"*
   - ✋ **체크포인트 ①**: 사용자 답변 입력 대기
   - 톤 브리핑 출력 (용도/시사점/톤/분량)

3. **아트디렉터** (`art-director` 에이전트) — 3옵션 제시
   - 콘솔 출력: 감성 이름 × 인터랙션 3가지
   - 편집장이 정리해서 콘솔에 표시

4. ✋ **체크포인트 ②**: 옵션 선택 입력 대기 (①②③)

5. **카피** (`copywriter` 에이전트) — 카피 작성
   - 오디언스/시사점 기반
   - 필요시 콘텐츠 보충 제안 (사용자 응답 받음)

6. **아트디렉터** (재호출) — 디자인 구현
   - 선택된 스타일로 HTML/CSS/Swiper
   - 필요시 OpenAI API로 이미지 생성 (`OPENAI_API_KEY` 사용)
   - `output/YYYY-MM/<slug>/index.html` (+ `mobile.html`)
   - 이미지는 `assets/YYYY-MM/<slug>/`

7. **편집장** — 최종 컨펌 체크리스트
   - ✅ 카피 오디언스/시사점 반영
   - ✅ 디자인 [스타일]×[인터랙션]
   - ✅ 모바일 버전 포함
   - ✅ 배포 전 이상 없음

8. ✋ **체크포인트 ③**: 컨펌 입력 대기

9. **편집장** — Vercel 배포
   - `vercel --prod` (또는 동등한 배포 명령)
   - 배포 URL 콘솔 출력

## 필수 환경변수 (`.env`)

```
OPENAI_API_KEY=<OpenAI 키>
```

> 텔레그램 봇 토큰은 이 커맨드에선 **불필요**. 그룹방 사용 안 하므로.

---

## 실행 전 체크리스트

- [ ] `.env` 에 `OPENAI_API_KEY` 채워져 있음
- [ ] 소스 파일이 `sources/YYYY-MM/` 아래에 있음
- [ ] 노션 URL 필요하면 소스 frontmatter에 `notion_url:` 추가됨
- [ ] `design.md` 에 `current_style` 설정됨
- [ ] `designs/<current_style>.md` 파일 존재

## 실패 시

| 단계 | 실패 케이스 | 복구 |
|---|---|---|
| 소스 파싱 | frontmatter 파싱 실패 | 수동으로 `---` 위치 확인 요청 |
| OpenAI 이미지 생성 | 1회 재시도 | 재시도도 실패 시 "이미지 0장으로 진행할까요?" 물음 |
| Vercel 배포 | 경로 오류 | 이미지 경로 확인 → 재시도 |

## 텔레그램 모드와의 차이점

| 항목 | 텔레그램 | 슬래시 커맨드 |
|---|---|---|
| 입력 | 그룹방 메시지 | CLI 인자 |
| 체크포인트 응답 | 그룹방 답장 | 콘솔 입력 |
| 봇 발화 | 각 봇이 그룹방에 메시지 | 콘솔 출력 (`[편집장]`, `[카피]`, `[아트]` 프리픽스) |
| 배포 URL 공유 | 그룹방에 편집장봇이 게시 | 콘솔 출력 |
| 로그 보관 | 텔레그램 채팅 기록 | 실행 로그 파일 (선택) |
