# 카드뉴스 자동화 공장 — 설계 플랜 v4 (상세판)

> 텔레그램 그룹방에 `.md` 소스(또는 텍스트)를 던지면 → **3개의 AI 봇 팀**이 그룹방에서 대화하듯 작업하고 → 편집장이 Vercel에 자동 배포하는 공장.
>
> **핵심 철학**: *오디언스와 시사점 없이 카피 안 쓴다.* 시사점 없는 콘텐츠는 생성하지 않는다. 이게 다른 AI 툴과의 차별점.

**개정 이력**
- v3 (2026-04-15): 1호 시제품 · 3봇 · 하이브리드(Claude Code) · Vercel 배포
- **v4 (2026-04-16 저녁)**: Phase 2 Node.js 폴링 런타임 + 카피 우선 합의 + 카피 A/B 비교(Opus+GPT-5) + 파일 첨부 기반 전달 + QA 루프 + 장표별 수정 메모 누적 + 세션 영속화 + 카피라이팅 가이드라인

## 1. Context

### 1-1. 왜 만드나

**현재 상태 (수동)**
- 매월 컨텐츠를 노션에 정리
- **젠스파크 / 캔바**로 카드뉴스 수동 제작
- 노션에 "홈페이지 게시" 해서 원문 URL 확보
- 결과물 공유

**문제점**
- 편당 디자인 선택, 슬라이드 분할, 카피 작성, 이미지 만들기 등 반복
- 도구가 좋아도 결국 사람이 다 손봄
- **결정적**: 다른 AI 툴은 "그냥 예쁘게" 만드는데 **누구한테 뭘 시키려는지** 반영이 안 됨

**가고 싶은 모습**
- `.md` + 노션 URL 한 줄만 던지면 → 3봇이 오디언스/시사점부터 정의 → 카피 합의 → 스타일 합의 → 최종 승인 → Vercel 배포
- 주현대리는 체크포인트 5번만 응답 (①, ①.5a, ①.5b, ②, ③)

### 1-2. 원하는 결과

| 단계 | 결과 |
|---|---|
| **v3 — 1호 시제품** (완료) | `경영진 AI 인사이트 4선` (15장) Vercel 배포 완주, 재사용 가능한 골격 확보 |
| **v4 — 공장화 (현재)** | 텔레그램 그룹방에서 완결. 파일 첨부 기반 검수 + Vercel 배포는 최종 1개만 |

**입력 단위**: source.md 1개 = 카드뉴스 1편. 편집장이 알아서 슬라이드 분할.
**입력 채널**: 텔레그램 그룹방. .md 첨부 또는 본문 텍스트.

## 3. 워크플로우 흐름도 (v4)

### 3-0. 시작 방식

**방식 A**: `.md` 파일 첨부 + frontmatter `notion_url:` → 편집장 바로 작업 시작
**방식 B (자연스러움) ⭐**: "카드뉴스 만들자" → 편집장이 본문·URL 순서대로 받음
**방식 C**: 긴 본문 텍스트 붙여넣기 (500자 이상 자동 감지)

```
                              ✋ 체크포인트 ③ "최종 선택"
                                  • 장표별 수정 메모 누적
                                  • "반영해" → 해당 옵션만 v2
                                  • "① v1 다시 봐" → 예전 버전 재전송
                                  • "① 컨펌" → 배포 (명시 키워드 필수)
                                  ⚠️ "1번으로 하자" 같은 모호 표현 ≠ 배포
                                                ↓
                                       🚀 [편집장] Vercel 1개 배포
                                       "🎉 https://cardnews-<slug>.vercel.app"


🎁 (가끔) "이 레포 좋아 보여" + 링크 ──→ designs/<새이름>.md 추가
```

### 3-1b. 상태 머신 요약

```
IDLE
  ↓ 소스 수신
AWAITING_AUDIENCE              ✋ 체크포인트 ①
  ↓ 오디언스 답변 + 톤 브리핑 + 카피 A/B 병렬 생성
AWAITING_COPY_APPROVAL         ✋ 체크포인트 ①.5
  ├ ①.5a A/B 선택 ("A"/"B"/"Opus"/"GPT")
  └ ①.5b 수정 메모 누적 → "반영해"(v2) → "카피 OK"(승인)
  ↓ "카피 OK"
AWAITING_OPTION                ✋ 체크포인트 ② (스타일 복수 선택)
  ↓ 스타일 N개 선택
BUILDING_FULLS                 카피 재사용 + 아트 병렬 + QA 루프
  ↓
AWAITING_FINAL_CHOICE          ✋ 체크포인트 ③
  ├ 장표별 수정 메모 누적
  ├ "반영해"(v2) / "① v1 다시 봐"(rollback)
  └ "① 컨펌"(명시적 배포 키워드)
  ↓ 명시적 컨펌
DEPLOYING → IDLE
```

## 7. 체크포인트 응답 양식

**원칙**: 자연어가 기본. 배포는 명시적 키워드 필수.

### 자연어 응답
- ① 오디언스: *"경영진"*, *"HR 임원"*, *"일반 직원"*
- ①.5a A/B: *"A"* / *"B"* / *"Opus"* / *"GPT"* / *"왼쪽"* / *"오른쪽"*
- ①.5b 카피 합의: 수정 메모 자유로이 → *"반영해"* (v2) → *"카피 OK"* (승인)
- ② 스타일: *"①"*, *"①③"*, *"셋 다"*, *"①③ 다시 뽑아"*
- ③ 최종: 수정 메모 누적 → *"반영해"* (v2) → *"① 컨펌"* (배포)

### 키워드 명령

| 키워드 | 의미 |
|---|---|
| `A` / `B` / `Opus` / `GPT` | 카피 v1 A/B 선택 |
| `카피 OK` / `승인` / `진행` | 카피 승인 → 아트 단계 |
| `반영해` / `v2 뽑아` / `다시 만들어` | 누적 메모 반영해서 새 버전 |
| `① 컨펌` / `② 배포해` / `확정` / `올려` / `푸시` | **최종 Vercel 배포** (명시 키워드 필수) |
| `① v1 다시 봐` | 예전 버전 파일 재전송 |
| `①③` / `셋 다` | 복수 스타일 선택 |
| `①③ 다시 뽑아` | 해당 옵션 재설계 |
| `취소` / `리셋` / `초기화` / `처음부터` | 세션 종료 (BUILDING 중에도 가능) |

### 배포 오인식 방지 ⚠️
- *"1번으로 하자"*, *"1번 좋아"* 같은 모호 표현 → **배포되지 않음**
- **반드시** `컨펌/확정/배포/올려/푸시/공유/OK(대문자)` 포함해야 Vercel 푸시
- 이유: 수정 의도로 보낸 메시지가 배포로 오인되는 사고 방지 (2026-04-16 오사고 반영)

### 모호한 응답 처리
봇이 100% 이해 못하면 추측 X, **다시 묻기**. 1회 재질문 후에도 모호하면 키워드 양식 안내.

## 12. FAQ — 왜 이렇게?

### Q1. 왜 카피를 A/B 두 버전으로?
- Opus와 GPT-5 는 관점·어휘·리듬이 다름. 어느 쪽이 주현대리 의도에 맞는지 **직접 비교**해서 고르는 게 가장 정확.
- 비용은 v1 만 2배. 이후는 선택된 한쪽만 호출 → 총 비용 증가폭 작음.

### Q2. 왜 카피를 스타일보다 먼저?
- 스타일은 **내용을 담는 그릇**. 내용 모르고 그릇부터 고르면 뒤집을 때 손실 큼.
- 카피 합의 → 스타일 → 풀 제작 순서면 롤백 비용이 단계적으로 격리됨.

### Q3. 왜 파일 첨부? Vercel은 느려?
- Vercel 무료 플랜 프로젝트 200개 한도. 매 preview = 새 프로젝트면 10편 만에 소진.
- 파일 첨부는 한도 없음, 배포 0초 (즉시 첨부), 비용 0.
- **최종 1개만 Vercel** 배포 → 프로젝트 1개/카드뉴스편.

### Q4. 왜 체크포인트 5개?
- 결정적 갈림길에만: ① 오디언스 (시작점) / ①.5 카피 (내용) / ② 스타일 (디자인) / ③ 최종 (배포).
- 각각 되돌리기 비용이 다름. 늦게 발견할수록 손실 큼 → 빠른 단계에 체크포인트.

### Q5. "1번으로 하자" 가 왜 배포 안 돼?
- 모호 표현. 수정 의도일 수도, 배포 의도일 수도 있음.
- 배포는 destructive (공개 URL 생성 + 프로젝트 카운트). 실수 방지가 가치 큼.
- **명시 키워드 필수** → 오인식 0.

### Q6. QA 루프가 왜 필요?
- LLM 출력은 가끔 문장 잘림·챕터 인디케이터 누락·타이포 불균형·인터랙션 코드 일부만 생성 등 흠 있음.
- 아트 셀프 QA + 편집장 체크 → 주현대리에게 가기 전에 내부에서 걸러냄.
- 비용 3배, 품질 안정성 ↑.

### Q7. 모델이 왜 전부 Opus?
- 카피·시사점은 품질이 차별점. 파싱도 Opus면 의도 해석 정확도 ↑.
- 비용은 Sonnet 대비 5배지만, 카드뉴스 1편당 전체 비용은 여전히 $5 이내.

### Q8. 한 카피라이터 페르소나가 두 LLM에 똑같이 적용되나?
- 네. `loadAgentSystem('copywriter')` 가 Opus·GPT 양쪽 system prompt 에 **동일하게** 주입.
- 차이는 모델 자체의 성향뿐. 공정한 A/B 비교.

## 14. 카피라이팅 가이드라인 (v4 신규)

> 카피라이터 봇 system prompt 에 주입. `.CLAUDE/agents/copywriter.md` 에만 포함. CLAUDE.md 전역은 건드리지 않음.
> Opus·GPT-5 **양쪽** 모두 이 가이드라인 적용.

### 핵심 원칙
**"경영진이 듣고 바로 움직이게 만드는 카피"**
감성으로 끌어당기고 → 이성으로 설득하고 → 실행으로 마무리.

### 분량 규칙
- 총 **최대 12장** (챕터 4개 × 챕터당 3장)
- **말줄임표(...) 절대 사용 금지**

### 슬라이드 제목 구조 (필수)
챕터 표지에는 반드시 **두 가지 모두** 작성:
1. **챕터 타이틀**: 주제가 명확히 보이는 제목
   - 예) `"AI Agent 시장 패러다임 변화"`
2. **슬라이드 헤드라인**: 강렬한 한 줄 카피
   - 예) `"AI 에이전트, 이제 IT팀 없이도 업무에 투입된다"`

### 헤드라인 작성법
- ❌ 숫자·사실 나열
- ✅ 사람이 있는 **극적인 장면**
- 좋은 예:
  - `"공동 창업자가 도망간 다음 날, 진짜 혁신이 시작됐다"`
  - `"3시간 만에 게임을 만들어 월 1.2억 원을 번 사람은 개발자가 아니었다"`

### 데이터 작성법
**수치 + 의미 + 맥락을 반드시 세트로**.
- 예) `"OpenClaw: 316,000★ — 역사상 최단기간 기록. 생태계를 빠르게 흡수 중"`

영어 키워드 사용 시 **한국어 풀이 병기**:
- 예) `"Ghost Growth: 이익↑ 소득↓ — 일자리 축소로 임금소득 공백 발생"`

### 시사점 작성법
- ❌ 추상적 인사이트
- ✅ **실행 가능한 구체적 액션**
- 예:
  - `"지금 할 일: HR×IT 공동으로 PoC 기준 먼저 만든다"`
  - `"60일 내 핵심 프로세스 3개 자동화 파일럿 가동"`

### CTA 작성법
- ❌ 홍보성 문구 — "워크숍 신청하세요"
- ✅ **메시지로 마무리**
- 예) `"기술은 이미 준비됐다. 조직이 준비되지 않았을 뿐이다."`

## 15. 모델 분배 상세 (v4 신규)

### 설정 (`bot-runtime/src/config.js`)
```js
models = {
  light: 'claude-haiku-4-5-20251001',  // 정의만, 미사용
  main:  'claude-opus-4-6',             // 카피·파싱·편집장·아트 옵션·QA
  heavy: 'claude-opus-4-6',             // 풀 HTML 생성
  gpt:   'gpt-5',                       // 카피 B (A/B 비교 전용)
}
```

### 함수별 호출 모델

| 함수 | 모델 | 역할 |
|---|---|---|
| `parseAudience` | Opus | 오디언스 답변 파싱 |
| `parseOptionChoice` | Opus | 스타일 옵션 파싱 (복수 + regenerate) |
| `parseFinalChoice` | Opus | 최종 단계 action 분류 (confirm/revise/rollback/select_variant 등) |
| `parseConfirmChoice` | Opus | 레거시 confirm 파서 |
| `conversationalRoute` | Opus | 세션 없을 때 자연 대화 라우터 |
| `handleSourceReceived` (오디언스 질문) | Opus | 편집장 발화 |
| `handleAudienceAnswer` (톤 브리핑) | Opus | 편집장 발화 |
| **`_generateCopyJson`** | **Opus** | **카피 A 생성 + v2~ 재생성** |
| **`_generateCopyJsonOpenAI`** | **GPT-5** | **카피 B 생성 (v1만)** |
| `proposeArtOptions` (3옵션 제안) | Opus | 아트디렉터 |
| `proposeArtOptions` (편집장 추천 요약) | Opus | 편집장 발화 |
| `generateOptionPreviews` | Opus | 미니 프리뷰 HTML (3슬라이드) |
| `_runStyleBuild` HTML v0 | Opus (heavy) | 풀 HTML 최초 생성 |
| `_runStyleBuild` 아트 QA | Opus (heavy) | 셀프 리뷰 + 수정 |
| `_runStyleBuild` 편집장 체크 | Opus | approve/issues 판단 |
| `_runStyleBuild` 재수정 | Opus (heavy) | 편집장 issues 반영 |
| `handleOptionRegenerate` | Opus | 옵션 재제안 |
| `answerQuestionsAndReprompt` | Opus | 편집장 질문 응답 |

### 카피 A/B 흐름
```
AWAITING_COPY_APPROVAL 진입 (v1)
   ↓
Promise.allSettled([
  _generateCopyJson(s),        ← Claude Opus
  _generateCopyJsonOpenAI(s),  ← GPT-5 (병렬)
])
   ↓ 약 60초 후 두 결과
copyDrafts = { opus, gpt }
   ↓ 주현대리 "A" 또는 "B" 선택
copyDraft = copyDrafts[chosen]
chosenProvider = 'opus' | 'gpt'
   ↓ v2~ 재생성은 chosenProvider 쪽만
```

### 비용·시간 영향

| 항목 | 값 |
|---|---|
| 카피 v1 생성 시간 | ~60초 (병렬이라 두 배 안 됨) |
| 카피 v1 비용 | 2배 (Opus + GPT-5 각각 1회) |
| 카피 v2~ 비용 | 1배 (선택된 한쪽만) |
| 풀 HTML 생성 | 스타일당 최대 4 LLM 호출 (생성 + 아트 QA + 편집장 + 재수정) |
| 카드뉴스 1편 전체 비용 | 약 $3~$5 |

### Graceful Degrade
- 한쪽 실패 → 성공한 쪽 단일 모드 진행
- 양쪽 실패 → 세션 리셋 + 에러 알림
