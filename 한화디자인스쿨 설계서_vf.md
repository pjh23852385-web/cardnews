# 카드뉴스 자동화 공장 — 설계 플랜 (상세판 v3)

> 텔레그램 그룹방에 `.md` 소스(또는 텍스트)를 던지면 → **3개의 AI 봇 팀**이 그룹방에서 대화하듯 작업하고 → 편집장이 Vercel에 자동 배포하는 공장.
>
> **핵심 철학**: *오디언스와 시사점 없이 카피 안 쓴다.* 시사점 없는 콘텐츠는 생성하지 않는다. 이게 다른 AI 툴과의 차별점.

---

## 📖 이 문서는 누가, 왜 보나요?

### 읽는 사람
- **주현대리** (이 시스템의 운영자/사용자)

- AI 에이전트 오케스트레이션을 배우고 싶은 사람
- **3개월 뒤의 나** (다시 봐도 이해할 수 있게)

| 목적 | 읽을 곳 |
|---|---|

## 1. Context

### 1-1. 왜 만드나

**현재 상태 (수동)**
- 매월 컨텐츠를 노션에 정리.
- **젠스파크 / 캔바**로 카드뉴스를 수동 제작 (디자인·슬라이드·카피 다 직접).
- 노션에 "홈페이지 게시" 해서 원문 URL 확보.
- 결과물 공유.

**문제점**
- 편당 디자인 선택, 슬라이드 분할, 카피 작성, 이미지 만들기 등 반복.
- 도구가 좋아도 결국 사람이 다 손봄.
- **결정적**: 다른 AI 툴은 "그냥 예쁘게" 만드는데, **누구한테 뭘 시키려는지** 반영이 안 됨.

**가고 싶은 모습**
- `.md` + 노션 URL **한 줄**만 던지면 → 3봇이 **오디언스/시사점부터 정의**한 후 카드뉴스 만들고 Vercel 배포까지 끝.
- 주현대리는 체크포인트 3번만 응답.

### 1-2. 원하는 결과

| 단계 | 결과 |
|---|---|
| **1호 시제품 (오늘)** | `01. 카드뉴스 소스.md` 한 편 자동 생성 → Vercel 배포 완주 + 재사용 가능한 골격 확보 |
| **공장화 (다음 편부터)** | 텔레그램 그룹방에 .md(또는 텍스트) 던지면 → 3봇이 대화하듯 작업 → 편집장이 Vercel 배포 후 링크 회신. 주현대리는 체크포인트 3번만 |

**입력 단위**: source.md **1개 = 카드뉴스 1편**. 1편 안에 섹션 1~N개. 편집장이 알아서 슬라이드 분할.

**입력 채널**: 텔레그램 그룹방. .md 파일 첨부 또는 본문 텍스트 직접 입력 둘 다 OK.

### 1-3. 결정 사항 (확정)

| # | 항목 | 결정 |
|---|---|---|
| 1 | **자동화 수준** | B (체크포인트형 반자동) — 주현대리가 3번 확인 |
| 2 | **디자인 소스** | `awesome-design-md` (VoltAgent 레포) 외부 참고 + `designs/` 라이브러리화 |
| 3 | **1호 디자인 선택 방식** | 아트디렉터가 본문/오디언스 보고 **3옵션 감성적 이름으로 제시** → 주현대리 선택 |
| 4 | **노션 URL 처리** | 주현대리가 노션 게시 후 URL 복사 → 소스 .md frontmatter `notion_url:` 한 줄 입력. **Notion API 자동 조회 X** |
| 5 | **이미지 생성 도구** | OpenAI API (DALL-E 3 + gpt-image-1, 키 1개로 둘 다 호출). 아트디렉터가 직접 호출 |
| 6 | **이미지 결정 방식** | 아트디렉터가 종합 판단. **이미지 0장도 가능** (CSS 그라디언트만으로 충분하면 패스) |
| 7 | **API 키 관리** | `.env` 파일, `.gitignore`로 깃 제외. 키 값은 주현대리가 직접 입력 (**채팅에 안 적기**) |
| 8 | **에이전트 구성** | **3개 에이전트** (편집장 / 카피 / 아트디렉터). 각각 텔레그램 봇으로 그룹방에서 대화 |
| 9 | **체크포인트** | ① 오디언스 정의 / ② 3옵션 중 선택 / ③ 최종 컨펌 — **그룹방에서 직접 묻기** |
| 10 | **입출력 채널** | 텔레그램 그룹방. 입력 = .md 파일/텍스트, 출력 = Vercel 링크 |
| 11 | **봇 매핑 (확정)** | 편집장=`@HanwhaFinanceNewsBot`, 카피=`@hl_copywriter_bot`, 아트디렉터=`@hl_artdirector_bot` |
| 12 | **핵심 철학** | "오디언스와 시사점 없이 카피 안 쓴다" — 모든 작업의 출발점은 누구한테/뭘 시키려는가 |
| 13 | **카피 우선 합의** | 디자인 옵션 전에 **카피 전체 초안**부터 주현대리 승인 (체크포인트 ①.5). 아트는 승인된 카피 기반으로 옵션 제안 |
| 14 | **미리보기/완성본 전달** | Vercel 배포 아님. **Telegram 파일 첨부** (.html 다운로드 → 브라우저). Vercel은 최종 컨펌 1개만 |
| 15 | **복수 스타일 킵** | ①②③ 중 1~N개 동시 선택 후 풀 HTML 병렬 제작. 마음 변하면 예전 버전 재전송 가능 |
| 16 | **장표별 수정 메모** | 수정 요구는 메모로 누적(재생성 X). "반영해" 할 때 한 번에 v2 생성. 배포는 "컨펌" 키워드 명시 필수 |
| 17 | **QA 루프 (내부)** | 풀 HTML 생성 후 주현대리 전달 전, 아트 셀프 QA + 편집장 체크 → 문장잘림/챕터인디케이터/타이포통일/인터랙션 검증 |
| 18 | **세션 영속화** | `session.json` 으로 런타임 재시작 시에도 세션 유지. 2시간 타임아웃 |
| 19 | **모델 분배** | 라우터/대화: `claude-sonnet-4-6` / 분기 파싱: `claude-haiku-4-5` / HTML 생성: `claude-opus-4-6` |

## 3. 워크플로우 흐름도 (재설계)

### 3-0. 시작 방식 — 두 가지 모두 OK

**방식 A: 한 번에 던지기 (빠름)**
- `.md` 파일 첨부 + frontmatter `notion_url:` → 편집장이 바로 작업 시작

**방식 B: 대화형 시작 (자연스러움) ⭐**
- 주현대리: *"카드뉴스 만들자"*
- 편집장: *"좋습니다. 텍스트(md) 주세요. 파일 첨부하셔도 되고 여기 붙여넣으셔도 돼요."*
- 주현대리: (.md 첨부 또는 본문 붙여넣기)
- 편집장: *"받았어요. 노션 원문 URL도 박으실 거에요? (있으면 마지막에 '원문 보기' 버튼 박힘, 없으면 스킵)"*
- 주현대리: *"있어, https://..."* 또는 *"없어"* 또는 *"나중에 줄게"*
- 편집장: *"OK. 분석 들어갈게요."* → 오디언스 확인 단계로

**필수 vs 선택**:
| 입력 | 필수? | 없으면? |
|---|---|---|
| 본문 내용 | ✅ 필수 | 편집장이 *"본문 주세요"* 로 받음 |
| 노션 URL | ⚪ 선택 | "원문 보기" CTA 생략 |
| frontmatter 형식 | ⚪ 선택 | 본문 텍스트만 있어도 OK |

## 4. 핵심 철학 (모든 봇이 공유)

> **"오디언스와 시사점은 항상 한 세트다."**
> **"오디언스 없이 카피를 쓰지 않는다."**
> **"시사점 없는 카피는 요약이지 카피가 아니다."**
> **"이게 다른 AI 툴과의 차별점이다."**

### CLAUDE.md 맨 위에 박을 정확한 문구

빌드 시 `CLAUDE.md` 파일의 **최상단에 다음 섹션**이 들어간다 (토씨 그대로):

```markdown
## 핵심 철학
오디언스와 시사점은 항상 한 세트다.
오디언스 없이 카피를 쓰지 않는다.
시사점 없는 카피는 요약이지 카피가 아니다.
이게 다른 AI 툴과의 차별점이다.
```

### 세부 적용
- **편집장**: 소스 받으면 무조건 오디언스부터 확인 (생략 X)
- **카피**: 오디언스-시사점이 한 세트. 분리 안 함
- **아트디렉터**: 디자인도 오디언스에 맞춰 3옵션 제시 (경영진이면 신뢰감, 일반 직원이면 친근감)
- **모든 봇**: 작업 시작 전 CLAUDE.md 핵심 철학을 컨텍스트로 로드

### 왜 이게 차별점인가
다른 AI 툴(젠스파크/캔바/Gamma 등)은 "예쁜 카드뉴스"는 만들지만, **누구한테 뭘 시키려는지** 반영이 약함. 결과적으로 **요약**에 그치는 경우가 많음. 우리는 시작점을 다르게 잡음.

## 7. 체크포인트 응답 양식 (하이브리드)

**원칙**: 자연어가 기본, 키워드 명령도 인식. 봇이 이해 못하면 다시 묻기.

### 자연어 응답 (기본)
주현대리가 그냥 평소처럼 말하면 됨:
- ① 오디언스: *"경영진"*, *"사업부장급"*, *"일반 직원"*
- ①.5 카피 합의: *"3번 슬라이드 헤드 약해"* (메모 누적) → *"반영해"* (v2) → *"카피 OK"* (승인)
- ② 스타일 선택: *"①로 가자"*, *"①③ 둘 다"*, *"셋 다 뽑아"* (복수 선택 OK)
- ③ 최종 선택: *"① 1페이지 문장 잘림 고쳐"* (메모 누적) → *"반영해"* (v2) → *"① 컨펌"* (배포)

### 키워드 명령 (선택)
| 키워드 | 의미 | 예 |
|---|---|---|
| `카피 OK` / `승인` / `진행` | 카피 승인 → 아트 단계 | "카피 OK" |
| `반영해` / `v2 뽑아` / `다시 만들어` | 누적된 수정 메모 반영해서 새 버전 | "반영해" |
| `① 컨펌` / `② 배포해` / `확정` | **최종 Vercel 배포** (명시적 키워드 필수) | "① 컨펌" |
| `① v1 다시 봐` | 예전 버전 파일 재전송 | "① v1 다시 봐" |
| `①③` / `셋 다` | 복수 스타일 선택 | "①③ 풀 뽑아" |
| `①③ 다시 뽑아` | 해당 옵션 재설계 | "①③ 리디자인" |
| `취소` / `리셋` / `초기화` | 세션 종료 (BUILDING 중에도 가능) | "취소" |
| `네가 채워줘` / `위임` | 카피 보충 위임 | "네가 채워줘" |

### 배포 오인식 방지 (중요)
- *"1번으로 하자"*, *"1번 좋아"* 같은 모호 표현으로는 **배포되지 않음**
- **반드시** "컨펌/확정/배포/올려/푸시/공유" 키워드 포함해야 Vercel 푸시
- 이유: 주현대리가 수정 의도로 보낸 메시지가 배포로 오인되는 사고 방지

### 모호한 응답 처리 (확인 우선)
봇이 100% 이해 못하면 추측 X, **다시 묻기**:
- *"죄송한데, '경영진' 이라 하셨는데 C레벨인가요? 사업부장급인가요?"*
- 1회 재질문 후에도 모호하면 키워드 양식 안내

### 응답 시간 정책
- **30분 무응답** → 봇이 한 번 리마인드: *"@주현대리 체크포인트 ② 응답 부탁드려요."*
- **2시간 무응답** → 작업 일시 정지. 응답 시 재개.

### 수동 백업 모드
텔레그램 안 쓰고 싶을 땐 `/make-cardnews <파일>` 슬래시 커맨드로도 동일하게 실행 가능.

   - ① 오디언스? → "경영진" / "일반 직원" 등
   - ①.5 카피 합의? → 수정 쌓기 → "반영해" → "카피 OK"
   - ② 어느 스타일? → "①" / "①③" / "셋 다" (복수 OK)
   - ③ 최종 선택? → 수정 쌓기 → "반영해" → "① 컨펌" (명시적 배포)
4. **완성본 파일 다운로드해서 브라우저에서 열기** (미리보기·완성본 모두 .html 첨부로 옴)
   - PC에서 확인, 모바일에서도 확인
   - 수정할 거 장표별로 텔레그램에 쌓기
5. **최종 1개만 Vercel 배포** → 공유용 URL 회신

**선택: 디자인 바꾸고 싶을 때**
- `design.md` 한 줄 수정, 또는 그룹방에 "이번엔 wired로 가" 한 마디

**🎁 가끔: 좋은 디자인 레포 발견했을 때**
- 그룹방에 "이거 새로 발견함: https://github.com/..." 식으로 링크 던지기
- 편집장이 받아서 `designs/<name>.md` 생성

**선택: 봇 페르소나 튜닝**

- 또는 그룹방에 "편집장님, 좀 더 부드럽게 가요" 한 마디 → 봇이 페르소나 갱신 제안

## 12. FAQ — 왜 이렇게?

### Q1. 왜 봇을 3개로 줄였나요? 5개에서?
**역할 자연스러움 우선.** 디자이너+이미지 분리가 어색했음. 한 사람이 디자인 보고 이미지까지 책임지는 게 자연스럽고, 의사소통 비용도 줄어듦. 또 봇 5개 동시 발화하면 그룹방 가독성 떨어짐. 3봇이 골든 존.

### Q2. 왜 publisher를 따로 안 두고 편집장에 합쳤나요?
시작과 끝을 같은 사람이 책임지는 게 자연스러움. 편집장이 본문 받아 팀 가동시켰으니 마무리(배포)도 본인.

### Q3. 왜 텔레그램으로 했나요?
- **현장감**: 봇들이 그룹방에서 대화하는 게 실제 팀 회의 느낌
- **모바일 친화**: 어디서든 본문 던지고 체크포인트 응답 가능
- **로그 자동 보관**: 그룹방 내역이 자연스럽게 작업 이력

### Q4. 왜 "오디언스/시사점 없이 카피 안 쓴다"가 핵심 철학인가요?
다른 AI 툴들의 결과물이 "예쁜데 누구한테 뭘 시키는지 모름" 이라는 게 가장 큰 한계. 이걸 처음부터 정의하고 가는 게 차별점. 비용 들여 만들었는데 안 통하는 카드뉴스 = 실패.

### Q5. 체크포인트가 왜 3개? 왜 그 위치?
완전 자동(0개)은 결과 예측 어렵고, 너무 많으면(5+) 자동화 의미 X. 3개는 **결정적 갈림길**:
- ① 오디언스 (시작점 — 잘못 정의하면 다 망함)
- ② 옵션 선택 (방향 결정)
- ③ 최종 컨펌 (배포 직전)
콘텐츠 보충은 카피가 알아서 위임 가능하니 선택적.

### Q6. 디자인 스타일을 왜 별도 라이브러리(designs/)로?
디자인은 자주 바뀌고, 공통 규칙은 거의 안 바뀜. 분리하면:
- 새 스타일 추가 = `designs/<name>.md` 한 파일
- 이번 편 변경 = `design.md` 한 줄

### Q7. 왜 비용을 안 보여주나요?
주현대리가 컨셉/오디언스에 집중하도록. 비용은 한 편당 0.5달러도 안 됨. OpenAI 대시보드에서 따로 확인 가능.

### Q8. 카피가 콘텐츠를 보충한다는 게 무슨 뜻?
본문 보다가 *"이 부분에 사례 하나 있으면 훨씬 강해져요. 추가 내용 있으세요?"* 라고 묻고, 주현대리가 *"네가 찾아서 넣어"* 하면 카피가 자체적으로 보충. 단, 보충 출처는 명시 (오디언스가 신뢰할 수 있게).

### Q9. 아트디렉터가 디자인부터 이미지 생성까지 다 하면 한 봇에 너무 많은 거 아닌가?
오히려 **자연스러움**. 디자인 톤을 결정한 본인이 어떤 이미지가 어울릴지 가장 잘 앎. 외부 위탁(이미지봇)하면 톤 깨질 위험. 부담 크면 나중에 "이미지 전담"을 분리할 수 있지만, 1호엔 통합으로 충분.

### Q10. 노션 게시 안 한 편도 만들 수 있나요?
가능. frontmatter에서 `notion_url:` 빼면 마지막 슬라이드의 "원문 보기" CTA가 자동으로 빠짐.
